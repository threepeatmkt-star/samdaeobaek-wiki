# 뉴스·블로그 검색 루틴 (Monitoring Routine)

> [!takeaway] 내 관점 핵심
> - 이 루틴은 위키의 **[시장·경쟁] 카테고리에 자동으로 연료를 붓는 파이프라인**이다. 매번 돌릴 때마다 결과를 `raw/`에 넣고 "정리해줘"만 하면 위키가 갱신된다.
> - 핵심 가치: **경쟁사 신제품·시장 트렌드·규제 변화를 놓치지 않는 조기경보**. 특히 [[브랜드-모니터링-2026-06|잠백이의 일상 건기식 확장]]과 [[건기식-광고규제-과징금강화|규제 강화]]는 이 루틴으로 잡았다.

## 루틴이 하는 일
네이버 검색 API로 **자사·경쟁사 브랜드**와 **시장 키워드**의 뉴스/블로그를 주기적으로 수집해 텍스트 파일로 떨군다.

### 수집 대상 (현재 설정)
- **브랜드 모니터링**: 삼대오백(자사), 익스트림, 잠백이, 이그니스, 제로투히어로
  - (이전 추적 대상이던 김단단은 2026-06-17 제외. 과거 데이터는 [[브랜드-모니터링-2026-06]]에 잔존.)
- **시장 키워드**: 단백질 시장 / 프로틴 음료 / 건강기능식품 시장 / 헬스 보충제 / 다이어트 식품 트렌드
- **트렌드 링크**: 8000억 단백질 시장, 고함량 단백질 음료, 일동후디스 프로틴 쉐이크, 코스맥스 스낵형 건기식, 건기식 후기광고 과징금 등 큐레이션

### 출력 파일 (raw/로 들어옴)
| 파일 | 내용 |
|---|---|
| `*_브랜드검색-result.txt` / `-w2.txt` | 브랜드별 뉴스·블로그 목록 + 날짜/링크 |
| `*_시장키워드-market.txt` | 시장 키워드 뉴스 + 요약 desc |
| `*_트렌드링크-trendlinks.txt` | 핵심 산업 기사 큐레이션 |

### 인증
- 네이버 검색 API(Client ID/Secret) 사용. **키는 `raw/뉴스루틴_네이버api-키.txt`에 보관** (위키 본문에는 비노출).

## ⚙️ 자동 실행 (2026-06-17 구축)
- **스크립트**: `scripts/fetch-news.ps1` (+ 설정 `scripts/config.json`)
  - 브랜드/키워드를 네이버 뉴스·블로그에서 검색 → **전일 데이터만 필터** → `raw/{날짜}_뉴스루틴_*.txt` 저장.
  - PowerShell 5.1 한글 깨짐 방지 위해 스크립트는 **UTF-8 BOM**으로 저장(중요: 수정 후 BOM 유지).
- **스케줄**: Windows 작업 스케줄러 `삼대오백-뉴스루틴`, **평일(월~금) 10:00**.
  - **주말 보정**: 월요일 실행 시 금·토·일 3일치를 가져온다(주말 미실행으로 인한 누락 방지).
- **수동 실행/테스트**: PowerShell에서 `& "…\scripts\fetch-news.ps1"`
- **수집 후 정리**: 데이터가 `raw/`에 쌓이면 일별 모니터링 페이지(`모니터링/YYYY-MM-DD.md`)로 정리. (정리 자동화 여부는 사용자 설정에 따름)
- 추적 브랜드 메모: **제로투히어로**는 보충제가 아니라 헬스 **장비/액세서리** 브랜드 → [[00-모니터링-목차]] 참조.

## 운영 방식 (위키 연결)
1. 루틴을 돌려 결과 `.txt`를 **`raw/`에 날짜 접두어로** 넣는다. (예: `2026-06-16_뉴스루틴_...txt`)
2. Claude에게 **"뉴스루틴 자료 정리해줘"** 라고 한다.
3. Claude가:
   - 신규 동향 → [[단백질-건기식-시장동향-2026-06|시장 동향 페이지]] 갱신/신설
   - 브랜드 움직임 → [[브랜드-모니터링-2026-06|브랜드 모니터링 페이지]] 갱신/신설
   - 규제·법령 → [[건기식-광고규제-과징금강화|규제 페이지]] 갱신
   - 월이 바뀌면 새 월간 페이지(`...-2026-07`)를 만들고 이전 페이지와 교차링크
4. `index.md`와 `log.md` 갱신.

> 월간 스냅샷 방식: 같은 달 자료는 한 페이지에 누적, 새 달은 새 페이지 + 이전 달 링크. 추세 비교가 쉬워진다.
