---
description: raw/의 최신 뉴스루틴 수집 데이터를 일별 모니터링 페이지로 정리
---

`raw/` 폴더에서 아직 정리되지 않은 최신 `*_뉴스루틴_브랜드검색*.txt`(및 `*_뉴스루틴_시장키워드*.txt`) 파일을 읽고, `CLAUDE.md` 규칙에 따라 정리해줘.

1. **일별 모니터링 페이지**: 파일의 각 데이터 날짜별로 `모니터링/YYYY-MM-DD.md` 생성.
   - 이미 있는 날짜 파일은 **건너뛰고 새 날짜만** 만든다.
   - 형식은 기존 `모니터링/2026-06-16.md`와 동일: 맨 위 `> 관전포인트:` 한 줄 → 브랜드별 섹션.
   - **노이즈 제외**: 코스트코/트레이더스 할인글, 무관한 검색 결과(동명 제약사 등)는 빼고 실제 브랜드 언급만.
   - 추적 브랜드: **삼대오백**(자사), **익스트림·잠백이·이그니스**(식품 경쟁), **제로투히어로**(장비 → MXSSIVE 경쟁).
   - 같은 SKU 후기가 여러 건이면 제품별로 묶어 대표 링크만.

2. **시장 동향**: 시장키워드 파일에 의미 있는 신규 동향이 있으면 `wiki/단백질-건기식-시장동향-2026-MM.md`에 반영(같은 달 누적, 새 달이면 신설 + 이전 달 교차링크). 규제·법령 이슈는 `wiki/건기식-광고규제-과징금강화.md` 갱신.

3. 🛰️ **경쟁사 레이더 (자율 확장)**: 추적 대상 외에 우리 카테고리(헬스식품·헬스장비·여성 웰니스)에 새 브랜드/플레이어가 보이면 — 카테고리가 분명하면 **승인 없이 `scripts/config.json`에 추가해 추적 시작**, 애매하면 `wiki/경쟁사-워치리스트.md` 후보 큐에 적재. 추가 내역은 워치리스트·log에 남긴다.

4. 📡 **트렌드 레이더 (과감하게)**: 핫한 **브랜딩 공법·마케팅 방식·신원료/신제형**이 보이면 `wiki/트렌드-레이더-2026-MM.md`에 적재(같은 달 누적, 새 달 신설+이전 달 교차링크)하고, 각 항목에 **'내 사업 적용(시도 아이디어)'**을 붙인다.

5. **목차·로그**: `모니터링/00-모니터링-목차.md` '일별 로그'에 새 날짜 링크 추가(+신호/추적대상 변화 시 상단 표 갱신), `wiki/index.md`·`wiki/log.md` 갱신.

6. **요약 + 💡제안**: 끝나면 ① 새로 만든/수정한 페이지 목록, ② 비즈니스 핵심 인사이트 1~2개, ③ **"💡 이번 회차 제안"**(자율 추가한 경쟁사·트렌드 + 사용자 판단이 필요한 시도 아이디어)을 한국어로 요약한다.

⚠️ 네이버 API 키 등 시크릿은 절대 위키 본문에 쓰지 마.
교차참조(`[[위키링크]]`)를 적극적으로 걸어줘.
